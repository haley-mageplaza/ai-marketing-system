# Blog Content Rules

## 1. Purpose

This document defines the **standard operating rules** for creating, auditing, and optimizing blog content across all brands (Mageplaza, Mageplaza Vietnam, Shopplaza).

This file is used by:
- Claude Code (for content generation & auditing)
- Human editors (for review & QA)
- SEO/content teams (for consistency)

The goal is to ensure all blog content:
- Matches **search intent precisely**
- Delivers **real value to readers**
- Is optimized for **SEO, AIO (AI Overviews), and GEO (Generative Engines)**
- Supports **internal linking and conversion funnels**
- Maintains **brand consistency**

---

## 2. Core Philosophy

### 2.1 Content exists to solve problems

Every blog must answer:
> “What problem does this content solve, and for whom?”

If the answer is unclear → the content is invalid.

---

### 2.2 Value > Volume

- Do not write to fill word count
- Do not expand content without adding value
- Every section must justify its existence

---

### 2.3 Clarity > Cleverness

- Prioritize understanding over creativity
- Avoid vague, abstract, or overly “marketing” language
- Write like explaining to a smart but busy person

---

### 2.4 Intent is the foundation

Every blog must clearly define:
- Primary keyword
- Search intent
- Target audience
- Funnel stage

Without this → do NOT generate content

---

## 3. Search Intent Framework

Claude must classify intent BEFORE writing.

### 3.1 Intent Types

| Type | Description | Content Style |
|------|------------|--------------|
| Informational | Learn / understand | Guides, definitions |
| Commercial | Compare / evaluate | Lists, comparisons |
| Transactional | Take action | Product/service focus |
| Navigational | Find specific page | Minimal content |

---

### 3.2 Intent Matching Rules

Content MUST:
- Directly answer what user is searching
- Avoid mixing unrelated intents
- Align structure with intent

Example:
- “Best Shopify B2B apps” → comparison format
- “What is Hyvä theme” → definition + explanation

---

## 4. Content Structure System

Every blog MUST follow this structure.

---

### 4.1 Introduction

Purpose:
- Hook attention
- Define problem
- Set expectations

Must include:
- Clear problem or question
- Why it matters
- What reader will learn

Avoid:
- Generic opening
- Long storytelling
- Keyword stuffing

---

### 4.2 Main Body (H2 / H3)

Each section must:
- Focus on ONE idea
- Answer a specific question
- Provide explanation + context

Structure:
- H2 = main topic
- H3 = sub-details

Each section should:
- Be self-contained
- Be skimmable
- Have logical flow

---

### 4.3 Content Depth

Content must include:
- Explanation (what)
- Reasoning (why)
- Application (how)

At least one of:
- Examples
- Use cases
- Comparisons
- Steps

---

### 4.4 Actionable Layer

Every blog must provide:
- Clear next steps OR
- Practical insights OR
- Decision guidance

---

### 4.5 Conclusion

Must:
- Reinforce key takeaway
- Be concise
- Optional CTA

Avoid:
- Repeating entire article
- Adding new ideas

---

## 5. SEO Rules

### 5.1 Keyword Strategy

Primary keyword:
- Title
- H1
- First 100 words
- 1–2 H2 headings

Secondary keywords:
- Naturally distributed

Avoid:
- Keyword stuffing
- Forced repetition

---

### 5.2 Heading Optimization

Headings must:
- Reflect real search queries
- Be descriptive and clear

Bad:
> Benefits of using it

Good:
> Benefits of using Hyvä theme for Magento stores

---

### 5.3 Topical Coverage

Content must:
- Cover main topic fully
- Include related subtopics
- Answer follow-up questions

---

### 5.4 Internal Linking

Each blog MUST include:
- 2–5 internal links minimum

Types:
- Cluster articles
- Product/service pages

Rules:
- Natural placement
- Contextual anchor text
- No generic anchors (“click here”)

---

### 5.5 Meta Elements

Each blog must include:
- Meta title (clickable, keyword-aligned)
- Meta description (value-driven)

---

## 6. AIO (AI Optimization) Rules

Content must be structured for AI summaries (Google AI Overviews).

---

### 6.1 Direct Answer Blocks

For key questions:
- Answer immediately (1–2 sentences)
- Then expand if needed

---

### 6.2 Definition Sections

Include clear definitions:
- “What is X”
- “How does X work”

---

### 6.3 Structured Information

Prefer:
- Bullet lists
- Steps
- Tables
- Comparisons

---

### 6.4 FAQ Section

Include 3–6 FAQs:
- Based on real search queries
- Clear, concise answers

---

## 7. GEO (Generative Engine Optimization)

Content must be usable by AI systems (ChatGPT, Gemini, etc.)

---

### 7.1 Write like a source

Content must:
- Be factual
- Be structured
- Be easy to quote

---

### 7.2 Entity Clarity

Always clearly define:
- Tools
- Platforms
- Concepts

Avoid ambiguity.

---

### 7.3 Context + Comparison

Include:
- Alternatives
- Trade-offs
- Comparisons

---

### 7.4 Reusability

Content should be:
- Extractable into summaries
- Usable in answers
- Logically structured

---

## 8. Readability Standards

- Short sentences
- Simple vocabulary
- Clear explanations

Avoid:
- Jargon without explanation
- Long paragraphs (>4 lines)

---

## 9. Tone & Style

Must be:
- Professional
- Conversational
- Direct

Avoid:
- Buzzwords
- Over-promising
- Sales-heavy tone

---

## 10. Funnel Alignment

Every blog must map to a funnel stage:

| Stage | Goal |
|------|------|
| TOFU | Educate |
| MOFU | Compare / guide |
| BOFU | Convert |

Content must:
- Match funnel intent
- Support next step

---

## 11. Content Quality Checklist

Before publishing, ensure:

### Structure
- Clear intro
- Logical flow
- Proper headings

### SEO
- Correct keyword placement
- No stuffing

### Value
- Real insights
- Not generic

### Linking
- Internal links included
- Contextual anchors

### AIO/GEO
- Answer blocks
- Structured sections
- FAQ included

---

## 12. Common Mistakes (Strictly Avoid)

- Writing without intent
- Thin content
- Repetitive content
- Forced internal links
- Keyword stuffing
- Overly long paragraphs
- Vague explanations

---

## 13. Output Standard (For Claude)

Every generated blog MUST include:

1. Title  
2. Meta title  
3. Meta description  
4. Introduction  
5. Full body (H2/H3 structure)  
6. FAQ section  
7. Internal linking suggestions  

---

## 14. Enforcement Rules

Claude MUST:
- Refuse to generate content if intent is unclear
- Prioritize clarity over length
- Follow structure strictly
- Ensure content is actionable and structured

---

## 15. Final Principle

High-quality blog content must:

- Help users understand faster  
- Help search engines understand clearly  
- Help AI systems extract answers easily  

If any of these fail → content is not acceptable
