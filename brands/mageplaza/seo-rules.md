# SEO Rules & Guidelines
## Purpose
This document defines the SEO standards for all content, pages, and marketing tasks.  
It ensures consistency across:
- Search engine optimization (Google, Bing, etc.)
- AI search (AIO – AI Overviews, LLM-based answers)
- Generative Engine Optimization (GEO)
- User experience and conversion

All outputs from Claude MUST follow these rules unless explicitly overridden.

---

# 1. Core SEO Principles

## 1.1 Search Intent First (Critical Rule)
Every page MUST match a clear search intent:

- Informational → guides, definitions, tutorials
- Commercial → comparisons, best tools, alternatives
- Transactional → product pages, service pages
- Navigational → brand or product-specific queries

❗ Never mix multiple primary intents on one page.

---

## 1.2 Value Over Volume
Content must:
- Answer the query clearly and directly
- Provide unique insights, not generic summaries
- Avoid filler content and keyword stuffing

Rule:
> If a paragraph does not add new value → remove or rewrite

---

## 1.3 EEAT (Experience, Expertise, Authority, Trust)
All content should demonstrate:

- Real expertise (technical depth where needed)
- Practical experience (examples, use cases)
- Authority (clear positioning, confident tone)
- Trust (accurate claims, no exaggeration)

Include where relevant:
- Case references
- Real-world scenarios
- Specific tools, features, or workflows

---

## 1.4 Clarity & Readability
- Use simple, direct language
- Avoid vague statements
- Keep sentences concise
- Prefer structure over long paragraphs

---

# 2. Keyword Strategy

## 2.1 Keyword Types
Each page should include:

- Primary keyword (1 main target)
- Secondary keywords (3–5 variations)
- Semantic keywords (entities, related terms)
- Long-tail keywords (specific queries)

---

## 2.2 Keyword Placement

### Must include:
- Title (H1 or meta title)
- First 100 words
- At least one H2
- Naturally throughout content

### Avoid:
- Keyword stuffing
- Repeating exact keyword unnaturally

---

## 2.3 Semantic SEO (Very Important)
Content must cover the topic **completely**, not just the keyword.

Include:
- Related entities
- Variations in phrasing
- Supporting subtopics
- Contextual explanations

Example:
Instead of repeating "Magento 2 store locator", include:
- store finder
- location search
- Google Maps integration
- nearby store detection

---

# 3. Content Structure Rules

## 3.1 Standard Structure

Every page should follow:

- H1 (clear topic)
- Short introduction (hook + context)
- H2 sections (main topics)
- H3 sections (details/features/examples)
- FAQ section (if applicable)
- Conclusion / next step

---

## 3.2 Heading Rules

- H1: only one per page
- H2: main sections
- H3: supporting details
- Headings must be meaningful, not generic

❌ Bad: "Features"
✅ Good: "Key Features That Improve Store Performance"

---

## 3.3 Introduction Rules

The intro must:
- Address the problem or question
- Clearly state what the reader will get
- Include the primary keyword naturally

Avoid:
- Long storytelling
- Generic opening

---

## 3.4 Paragraph Rules

- 2–4 sentences per paragraph
- One idea per paragraph
- Use bullets when listing

---

# 4. On-Page SEO

## 4.1 Meta Title

- 50–60 characters
- Include primary keyword
- Add value or differentiation

Example:
"Magento 2 Store Locator: Features, Benefits & Setup Guide"

---

## 4.2 Meta Description

- 140–160 characters
- Clear value proposition
- Encourage click

---

## 4.3 URL Structure

- Short and readable
- Include primary keyword
- Avoid unnecessary parameters

Example:
`/magento-2-store-locator/`

---

## 4.4 Internal Linking (Critical)

Every page MUST:

- Link to:
  - At least 1 related article (cluster content)
  - At least 1 money page (product/service)

- Use natural anchor text
- Avoid generic anchors like "click here"

Example:
Instead of:
"learn more here"

Use:
"Magento 2 store pickup solution"

---

## 4.5 External Linking

Use when:
- Supporting claims
- Referencing tools or platforms
- Adding credibility

---

# 5. Content Depth & Coverage

## 5.1 Avoid Thin Content

A page must:
- Fully answer the topic
- Cover key subtopics
- Provide actionable insights

---

## 5.2 Add Real Value

Include:
- Examples
- Use cases
- Comparisons
- Pros & cons
- Practical steps

---

## 5.3 Avoid Generic AI Content

Do NOT:
- Repeat common definitions without adding value
- Write vague or surface-level content

---

# 6. AI SEO (AIO & GEO Optimization)

## 6.1 Answer-First Content
Content should include clear answers that AI can extract:

- Definitions
- Step-by-step instructions
- Direct explanations

---

## 6.2 Structured Answers

Use:
- Bullet points
- Numbered lists
- Clear sections

---

## 6.3 Entity Clarity

Clearly define:
- Product names
- Features
- Concepts
- Tools

Avoid ambiguity.

---

## 6.4 FAQ Optimization

Include FAQ section with:
- Real user questions
- Clear, concise answers

---

## 6.5 Snippet Optimization

Write sections that can be extracted as:
- Featured snippets
- AI summaries

---

# 7. Conversion-Oriented SEO

SEO content must also convert.

## 7.1 Clear CTA

Every page must have:
- At least one clear next step

Examples:
- Book consultation
- Try the app
- Explore features

---

## 7.2 Align with Funnel

- TOFU → educate
- MOFU → compare, explain
- BOFU → convert

---

## 7.3 Reduce Friction

Avoid:
- Confusing structure
- Overloaded text
- Missing context

---

# 8. Technical SEO Basics

Ensure:

- Fast loading speed
- Mobile-friendly layout
- Clean HTML structure
- Proper heading hierarchy

---

## 8.1 Image Optimization

- Use descriptive alt text
- Include keywords naturally
- Avoid keyword stuffing

---

## 8.2 Schema Markup (Recommended)

Use when relevant:
- FAQ schema
- Product schema
- Article schema
- Service schema

---

# 9. Content Quality Checklist

Before publishing, verify:

- Matches search intent
- Covers topic completely
- Includes internal links
- Uses clear structure
- Provides real value
- Optimized for AI extraction
- Includes CTA

---

# 10. What to Avoid

❌ Keyword stuffing  
❌ Duplicate content  
❌ Generic AI writing  
❌ Misleading claims  
❌ Over-optimization  
❌ Irrelevant internal links  

---

# 11. Final Rule

SEO is not about ranking only.

It must:
- Attract the right audience
- Deliver clear value
- Guide users to the next step

If content ranks but does not convert → it fails.
